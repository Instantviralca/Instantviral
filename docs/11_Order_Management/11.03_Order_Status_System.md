# InstantViral v2

# Document 11.03 — Order Status System

Version: 1.0

Status: Production Specification

---

# Purpose

This document defines the official order status system for InstantViral.

Every order must always have one valid status.

The same statuses must be used consistently across:

- Admin Panel
- Order Tracking
- Notifications
- Analytics
- Reports
- Future Customer Dashboard

---

# Objectives

The status system should:

- Be simple.
- Be predictable.
- Prevent invalid transitions.
- Keep customers informed.
- Support manual order processing.

---

# Official Statuses

Supported statuses:

- Pending
- Processing
- Completed
- Partial
- Cancelled
- Refunded (future)

No additional production statuses should be introduced without updating this document.

---

# Status Definitions

## Pending

Meaning:

- Order has been created.
- Payment has been verified.
- Waiting for manual review.

Customer sees:

"Your order has been received and is waiting to be processed."

## Processing

Meaning:

- Admin has started fulfilling the order.

Customer sees:

"Your order is currently being processed."

## Completed

Meaning:

- Order has been fully completed.

Customer sees:

"Your order has been completed successfully."

## Partial

Meaning:

- Only part of the order could be completed.

Customer sees:

"Your order was partially completed. Please contact support if you have questions."

## Cancelled

Meaning:

- Order will not be fulfilled.

Customer sees:

"Your order has been cancelled."

## Refunded (Future)

Meaning:

- Payment has been refunded.

Customer sees:

"Your order has been refunded."

---

# Allowed Status Transitions

```
Pending
↓
Processing
↓
Completed
```

Alternative paths:

- Pending → Cancelled
- Processing → Partial
- Processing → Cancelled
- Completed → Refunded
- Partial → Refunded

Cancelled and Refunded are terminal statuses.

---

# Invalid Transitions

The following transitions are not allowed:

- Completed → Processing
- Completed → Pending
- Cancelled → Processing
- Cancelled → Completed
- Refunded → Processing
- Refunded → Completed
- Pending → Completed (without review)

# Status Colors

| Status | Color |
|---|---|
| Pending | Yellow |
| Processing | Blue |
| Completed | Green |
| Partial | Orange |
| Cancelled | Red |
| Refunded | Gray |

These colors should use the Design System tokens rather than hardcoded values.

---

# Customer Visibility

Customers should only see:

- Current status
- Last updated time
- Optional public status message

Internal admin notes must never be exposed.

---

# Admin Actions

Depending on the current status:

**Pending**

- Start Processing
- Cancel

**Processing**

- Complete
- Mark Partial
- Cancel

**Completed**

- View
- Refund (future)

**Partial**

- View
- Refund (future)

**Cancelled**

- View

**Refunded**

- View

---

# Timeline

Every status change creates an immutable timeline entry containing:

- Previous status
- New status
- Timestamp
- Updated by
- Optional internal note

---

# Notifications

Automatically notify customers when:

- Order created
- Processing started
- Completed
- Partial
- Cancelled
- Refunded

Notification content is handled by the Notification System.

---

# Analytics

Track:

- Orders by status
- Average processing time
- Completion rate
- Cancellation rate
- Partial fulfillment rate

---

# Security

- Only authorized administrators may change order status.
- Every change must be logged.
- Status history cannot be deleted.

---

# Integration Points

Works with:

- Manual Order Workflow
- Notification System
- Order Tracking
- Admin Panel
- Analytics
