# InstantViral v2

# Document 08.00 — Homepage Blueprint

Version: 1.0

Status: Master Homepage Specification

---

# Purpose

This document defines the overall homepage architecture.

It is the master specification for every homepage section.

Individual section documents must follow this blueprint.

If a section conflicts with this document, this document takes precedence.

---

# Homepage Goals

The homepage has five objectives:

1. Explain InstantViral.
2. Build trust.
3. Help visitors find the correct platform.
4. Educate before selling.
5. Guide visitors toward conversion.

---

# Primary Audience

- Content creators
- Businesses
- Brands
- Agencies

---

# Homepage Success Criteria

Visitors should understand within the first few seconds:

- What InstantViral does.
- Which platforms are supported.
- Why InstantViral is trustworthy.
- Where they should go next.

---

# Homepage Information Hierarchy

The homepage should follow this order.

1. Hero
2. Trust Bar
3. Platform Grid
4. Services Grid
5. Why Choose InstantViral
6. How It Works
7. Statistics
8. Testimonials
9. FAQ
10. Final CTA
11. Footer

This order should not change unless there is a documented UX reason.

# Visual Rhythm

Alternate section backgrounds to improve readability.

Recommended pattern:

- Hero → Light
- Trust Bar → Slightly tinted
- Platform Grid → Light
- Services Grid → Neutral
- Why Choose Us → Light
- How It Works → Neutral
- Statistics → Brand accent
- Testimonials → Light
- FAQ → Neutral
- Final CTA → Brand accent
- Footer → Dark

Avoid long runs of identical backgrounds.

---

# Spacing System

Desktop:

- Section padding: 96–128px
- Content gap: 32–48px

Tablet:

- Section padding: 72–96px

Mobile:

- Section padding: 56–72px

Use the existing design tokens. Do not hardcode spacing.

---

# Layout Rules

- Maximum container width: 1280px
- 12-column grid on desktop
- Consistent vertical rhythm
- Mobile-first responsive design

---

# CTA Distribution

Primary CTA:

- Hero
- Final CTA

Secondary CTA:

- Services Grid
- Why Choose Us

Avoid placing a primary CTA in every section.

---

# Trust Signal Distribution

Trust should be reinforced throughout the page.

Examples:

- Hero badges
- Process transparency
- Testimonials
- FAQ answers
- Secure checkout messaging

Do not rely on a single "trust section."

---

# Internal Linking Strategy

Homepage should link to:

- Instagram
- TikTok
- YouTube
- Facebook

Each platform page should link to:

- Related services
- Learn articles
- FAQ

Articles should link back to:

- Relevant services
- Platform hubs

---

# Accessibility Standards

Every section must:

- Use semantic HTML
- Support keyboard navigation
- Maintain sufficient color contrast
- Provide meaningful alt text
- Avoid relying on color alone

---

# Performance Requirements

- Optimize all images with `next/image`
- Use WebP/AVIF where possible
- Lazy-load below-the-fold visuals
- Keep JavaScript minimal
- Avoid layout shift

---

# Animation Guidelines

Animations should be subtle and purposeful.

Allowed:

- Fade
- Slide
- Scale
- Stagger

Avoid:

- Bounce
- Flashing
- Continuous looping
- Distracting effects

---

# SEO Strategy

The homepage should target broad commercial intent.

Supportive sections should naturally introduce:

- Instagram
- TikTok
- YouTube
- Facebook

Avoid keyword stuffing.

---

# AI Search Strategy

The homepage should:

- Answer "What is InstantViral?" immediately.
- Use descriptive headings.
- Keep paragraphs concise.
- Support structured data.
- Link to deeper resources.

---

# Analytics

Track:

- Hero CTA clicks
- Platform selections
- Service selections
- Scroll depth
- FAQ interactions
- Final CTA clicks

---

# Component Rules

All homepage sections must:

- Use existing design system components.
- Read content from the data layer.
- Avoid hardcoded copy.
- Avoid inline styles.
- Be reusable and composable.

---

# Acceptance Checklist

- Homepage follows the documented section order.
- Consistent spacing across all sections.
- Responsive at all breakpoints.
- Clear conversion path.
- Trust signals distributed naturally.
- Strong internal linking.
- Matches Brand Strategy.
- Matches Design System.
- Meets accessibility requirements.
- Meets performance targets.
