# InstantViral v2

# Document 08.02 — Platform Selector

Version: 1.0

Status: Production Specification

---

# Purpose

The Platform Selector is the primary navigation section immediately after the Hero.

Its job is to help visitors quickly choose the social media platform they want to grow.

This section should reduce decision fatigue by presenting only the four supported platforms:

- Instagram
- TikTok
- YouTube
- Facebook

The section should act as a bridge between the homepage and the platform hub pages.

---

# Business Goals

- Increase clicks to platform pages.
- Help users self-identify their needs.
- Reduce homepage bounce rate.
- Improve internal linking.
- Guide visitors into the correct conversion path.

---

# UX Goals

Users should be able to answer one question instantly:

Which platform do you want to grow?

The design should make each platform equally easy to discover without implying one is more important than another.

---

# Search Intent

Primary intent:

Commercial Investigation

Secondary intent:

Informational

Users may not yet know which service they need, but they usually know which platform they care about.

---

# Section Heading

## H2

Choose Your Platform

## Intro Paragraph

Briefly explain that InstantViral offers services and educational resources for Instagram, TikTok, YouTube and Facebook.

Limit the introduction to 40–60 words.

---

# Layout

Desktop:

- 4 equal cards in one row.
- Responsive 12-column grid.
- Equal spacing.

Tablet:

- 2 × 2 grid.

Mobile:

- Single-column stack.

---

# Card Structure

Each platform card contains:

- Platform icon
- Platform name
- Short description (20–35 words)
- Top services preview (3–4 items)
- Primary CTA

# Card Behaviour

Hover:

- Slight elevation
- Soft shadow
- Border highlight
- Icon scale (subtle)

Focus:

- Visible focus ring
- Keyboard accessible

Active:

- Maintain accessible contrast

---

# Design Rules

- Equal card heights.
- Consistent padding.
- Icons sized consistently.
- Do not overload cards with text.
- Keep whitespace generous.

---

# Content Rules

All content must come from the content layer.

Do not hardcode:

- Titles
- Descriptions
- Services
- Links
- CTA labels

---

# SEO

The section should naturally reference all four platforms.

Use descriptive anchor text.

Avoid keyword stuffing.

---

# AI Search

The introductory paragraph should clearly explain what the section offers.

Each platform description should stand on its own if extracted by AI-powered search systems.

---

# Accessibility

- Semantic list or grid.
- Keyboard navigation.
- Descriptive link labels.
- Decorative icons marked appropriately.
- Screen-reader-friendly card structure.

---

# Performance

- Use SVG icons where possible.
- No heavy images required.
- Keep animations lightweight.
- Avoid unnecessary JavaScript.

---

# Analytics

Track:

- Platform card impressions.
- Platform CTA clicks.
- Card hover interactions (optional).
- Platform selection rate.

---

# Component Architecture

Create reusable components:

- PlatformSelectorSection
- PlatformGrid
- PlatformCard
- PlatformIcon
- PlatformServicesPreview
- PlatformCTA

All components must accept typed props.

---

# Data Source

Read from:

- `data/platforms.ts`
- `data/services.ts`
- `data/content/homepage.ts`

No duplicated platform information.

---

- AI-friendly descriptions.
- Consistent with Brand Strategy.
