# InstantViral v2

# Document 08.01 — Homepage Hero

Version: 1.0

Status: Production Specification

---

# Purpose

The Hero is the most important section of the homepage.

Within the first 5–8 seconds, visitors should understand:

- What InstantViral does.
- Which platforms are supported.
- Why they should trust InstantViral.
- What action they should take next.

The Hero should reduce uncertainty and guide users into the website—not overwhelm them with sales messaging.

---

# Business Goal

The Hero should:

- Increase trust immediately.
- Direct visitors to platform pages.
- Encourage exploration.
- Improve click-through rate to services.
- Support long-term conversions rather than short-term pressure.

---

# UX Goal

The Hero should answer four questions above the fold:

1. What is InstantViral?
2. Who is it for?
3. What platforms do you support?
4. What should I do next?

No scrolling should be required to understand the product.

---

# AI Overview Strategy

The Hero should clearly communicate:

- What the platform does.
- Which platforms it supports.
- Who it helps.
- Why it exists.

The introductory paragraph should be understandable even if shown independently by AI-powered search systems.

↓

↓

↓

↓

---

# Hero Illustration

Purpose:

Reinforce the platform visually.

Should not distract from the message.

Recommended content:

- Dashboard-style mockup
- Platform icons
- Analytics cards
- Growth charts
- Device mockup

Do not use generic stock photos.

---

# Image Specifications

Aspect Ratio:

16:10

Format:

WebP

Alt Text:

Describe the illustration accurately.

Priority:

Yes (LCP element).

Use `next/image`.

---

# Motion

Headline:

Fade up (0.4s)

Paragraph:

Fade up (0.5s)

Buttons:

Fade up (0.6s)

Illustration:

Scale + Fade (0.7s)

Hover:

Subtle lift only.

Avoid distracting animations.

---

# Accessibility

Use one H1 only.

Buttons must be keyboard accessible.

Maintain sufficient color contrast.

Visible focus states.

Meaningful alt text.

---

# Performance

Hero should contribute positively to Core Web Vitals.

Requirements:

- Optimized images
- Lazy-load non-critical assets
- Minimal JavaScript
- Avoid layout shift

---

# Analytics

Track:

- Hero primary CTA clicks
- Hero secondary CTA clicks
- Platform card clicks
- Scroll beyond Hero
- Time to first interaction

---

# Internal Linking

Primary CTA:

Platform selection section.

Secondary CTA:

Services overview.

Trust indicators should not be clickable.

---

# Cursor Implementation Notes

Build the Hero using existing design system components only.

Compose with:

- Container
- Section
- Stack
- Heading
- Text
- Button
- HeroImage
- Motion utilities

No inline styles.

No hardcoded colors.

No duplicated spacing values.

Content must come from `data/content/homepage.ts`.
