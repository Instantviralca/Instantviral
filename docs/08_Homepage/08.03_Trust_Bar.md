# InstantViral v2

# Document 08.03 — Trust Bar

Version: 1.0

Status: Production Specification

---

# Purpose

The Trust Bar reinforces confidence immediately after visitors choose a platform.

It should communicate reliability through clear, factual trust signals rather than marketing slogans.

This section is intentionally compact and should not distract from the primary user journey.

---

# Business Goals

- Increase user confidence.
- Reduce hesitation.
- Support conversion.
- Reinforce transparency.
- Improve perceived credibility.

---

# UX Goals

Within a few seconds, visitors should understand that InstantViral is:

- Easy to use
- Transparent
- Secure
- Helpful

The Trust Bar should scan quickly without requiring detailed reading.

---

# Layout

Desktop:

- Horizontal row of trust items.
- 4 equal columns.

Tablet:

- 2 × 2 grid.

Mobile:

- Single-column stack.

---

# Section Heading

No H2 required.

The Trust Bar supports the surrounding content and should remain visually lightweight.

---

# Trust Items

Recommended default items:

## Secure Checkout

Short description:

Payments are handled through secure and trusted checkout methods.

## Transparent Process

Short description:

Clear service information with straightforward ordering steps.

## Helpful Support

Short description:

Reach out whenever you need assistance before or after placing an order.

## Multiple Platforms

Short description:

Services and educational resources for Instagram, TikTok, YouTube and Facebook.

---

# Design Rules

Each trust item contains:

- Simple icon
- Short title
- One concise description

Equal card heights.

Minimal borders.

Subtle background.

No excessive decoration.

---

# Icons

Use Lucide React icons only.

Suggested icons:

- ShieldCheck
- BadgeCheck
- Headphones
- Layers3

Icons should use the existing design tokens.

---

# Typography

Title:

16–18px equivalent.

Description:

14–16px equivalent.

Descriptions should remain under two lines on desktop where possible.

---

# Animation

Entrance:

Fade up with slight stagger.

Hover:

Very subtle elevation.

No looping or attention-grabbing effects.

---

# Accessibility

- Icons should not be the only way information is conveyed.
- Titles must be text.
- Contrast must meet WCAG AA.
- Keyboard navigation should remain logical.

---

# SEO

This section should not target keywords directly.

Its purpose is UX and trust.

Avoid keyword stuffing.

---

# AI Search

Write descriptions so they are understandable if extracted independently.

Do not include unsupported claims.

---

# Performance

- SVG icons only.
- No images.
- Minimal JavaScript.
- Lightweight animations.

---

# Analytics

Track:

- Section impressions.
- Visibility percentage.
- Optional interaction with trust items if made clickable in the future.

---

# Component Architecture

Create:

- TrustBarSection
- TrustGrid
- TrustCard
- TrustIcon

Components must be reusable.

---

# Data Source

Read all trust items from:

`data/content/homepage.ts`

or

`data/trust.ts`

No hardcoded text inside components.

---

# Acceptance Checklist

- Four trust items.
- Fully responsive.
- Accessible.
- Data-driven.
- Uses existing design system.
- No hardcoded copy.
- Build passes.
- Consistent with Brand Strategy.
- Consistent with Homepage Blueprint.
