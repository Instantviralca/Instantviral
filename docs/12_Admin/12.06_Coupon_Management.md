# 12.06 Coupon Management

**Version:** 1.0  
**Module:** Admin  
**Status:** Planned

## 1. Purpose

Create and operate promotional coupon codes with deterministic eligibility, safe redemption limits, validity controls, and measurable results.

## 2. Route, Permissions, and Audit

- Route: `/admin/coupons` with editor route `/admin/coupons/[couponId]`.
- [ ] Require `settings.view`/`settings.manage` until dedicated `coupons.view`/`coupons.manage` permissions exist.
- [ ] Log creation, edits, activation/deactivation, deletion/archival, and manual exceptions with actor and before/after summary.
- [ ] Never modify a coupon snapshot already applied to a completed order.

## 3. Coupon Table

The table supports search by code/name and filters for status, discount type, campaign, validity window, service/platform scope, and redemption state.

| Column | Content |
| --- | --- |
| Code | Normalized displayed code with copy action. |
| Discount | Percentage or fixed amount, currency where applicable. |
| Scope | Eligible services, packages, platforms, and customer rules. |
| Validity | Starts, expires, timezone, and active status. |
| Usage | Redemptions, limit, and remaining capacity. |
| Performance | Discount granted, attributed revenue, and average order value. |
| Actions | Edit, duplicate, deactivate, archive, and view detail. |

## 4. Coupon Editor

### 4.1 Code and identity

Codes are unique after uppercase normalization and trimming. Reject ambiguous whitespace and unsafe characters. Support an internal campaign name and staff-only description separately from the customer-visible code.

### 4.2 Discount types

- Percentage discount: require a value greater than 0 and at most 100.
- Fixed amount: require a positive minor-unit amount and an eligible currency.
- [ ] Define rounding against the order subtotal using the checkout currency formatter.
- [ ] Define whether shipping, fees, tax, and already-discounted lines are excluded; do not leave calculation order ambiguous.
- [ ] Prevent a discount from reducing the payable total below zero.

### 4.3 Eligibility rules

Rules may constrain services, package IDs, platforms/categories, minimum subtotal, currency, first-order/customer segment, and allowed customer emails/domains when approved. Rules use AND semantics across rule groups and OR semantics within a selected list; the UI must explain the evaluated result.

- [ ] Validate package IDs against real catalog/service mappings.
- [ ] Define stacking policy explicitly; default is one coupon per order and no stacking with incompatible automatic discounts.
- [ ] Evaluate all rules server-side at price calculation and order placement.

### 4.4 Validity and limits

Configure start/end timestamps with explicit timezone, active state, total redemption limit, per-customer limit, and optional campaign budget. Expired, exhausted, inactive, and not-yet-active coupons are rejected with safe customer messages.

## 5. Validation and Redemption Safety

Coupon validation returns a stable, non-sensitive reason code for ineligibility and must be re-run when cart contents, currency, customer context, or time changes. Order placement atomically reserves or increments redemption capacity to prevent concurrent over-redemption. Failed or abandoned checkout attempts release any temporary reservation according to a documented timeout policy.

## 6. Coupon Statistics

The detail view reports redemptions, unique customers, conversion from valid attempt to order, gross sales, discount value, net revenue, average order value, first-order share, and redemptions over time. Date range and timezone are visible; refunds/cancellations are labeled and handled consistently in attribution.

- [ ] Link permitted aggregate metrics to filtered orders without exposing unnecessary customer data.
- [ ] Distinguish attempted, validated, applied, paid, refunded, and cancelled redemptions.
- [ ] Export only for authorized users and audit the export.

## 7. Acceptance Checklist

- [ ] Codes are normalized, unique, and safely validated server-side.
- [ ] Discount arithmetic, scope, stacking, validity, and limits are deterministic.
- [ ] Checkout revalidates and atomically applies coupon capacity.
- [ ] Historical orders retain their applied coupon snapshot.
- [ ] Staff can inspect clear, timezone-aware performance statistics.
