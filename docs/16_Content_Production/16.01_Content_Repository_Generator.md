# Document 16.01 — Content Repository Generator

Version: 1.0
Status: Production Specification

## Purpose

Create a file-based Content Repository for every Learn Center article.

## Goals

- One folder per article
- Consistent file structure
- Git-friendly
- MDX-first
- JSON for structured data
- Easy to review and update
- No database required

## Repository Structure

content/
├── instagram/
├── tiktok/
├── facebook/
├── youtube/
└── marketing/

Each article folder contains:

00_README.md
01_Research.md
02_Article_Brief.md
03_Article.mdx
04_SEO.json
05_FAQ.json
06_JSON-LD.json
07_Image_Brief.md
08_Internal_Links.md
09_External_Sources.md
10_Social_Posts.md
11_Email_Newsletter.md
12_Content_Checklist.md
13_Editor_Notes.md
14_Changelog.md

## Naming Rules

- Lowercase
- Hyphen-separated slugs
- One article per folder
- No duplicate slugs

## Required Scripts

- createArticlePackage()
- validateRepository()
- listMissingFiles()
- buildContentIndex()

## Validation

Check:

- Missing files
- Duplicate slugs
- Invalid metadata
- Empty article files
- Broken internal links
- Missing SEO assets

## Acceptance Checklist

- Repository created
- Folder template created
- Validation script prepared
- Naming rules documented
- Ready for all 50 articles
