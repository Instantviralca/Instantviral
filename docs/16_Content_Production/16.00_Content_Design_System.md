# Document 16.00 — Content Design System

## Purpose
Define one editorial standard for every Learn Center article.

## Tone of Voice
- Professional
- Helpful
- Honest
- Beginner-friendly
- No hype or exaggerated claims

## Article Structure
1. SEO Metadata
2. Hero
3. Quick Answer
4. Table of Contents
5. Introduction
6. Main Sections (H2/H3)
7. Key Takeaways
8. FAQ
9. Conclusion
10. CTA

## Heading Rules
- One H1
- Logical H2/H3 hierarchy
- Descriptive headings
- No skipped heading levels

## Callout Types
- Pro Tip
- Important
- Warning
- Example

## Tables
Use only when comparison improves understanding.

## Checklists
Use at article end for actionable summaries.

## CTA Rules
- Helpful, never pushy
- Max one in body
- One final CTA

## Internal Links
- 2–5 related articles
- 1–2 relevant service pages
- Category page
- Never force links

## External Sources
Prefer:
1. Official platform documentation
2. Primary research
3. Reputable publications

## Images
- Featured image
- Diagrams where useful
- Alt text required
- 1600x900 preferred

## FAQ Rules
- 8–15 useful questions
- Match visible content
- Schema only for visible FAQs

## SEO Rules
- Natural keyword usage
- Helpful-first writing
- No keyword stuffing
- Unique metadata

## Acceptance
Every article must follow this standard before publication.
