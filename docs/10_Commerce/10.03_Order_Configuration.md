# InstantViral v2

## Order Configuration

Version: 1.0

Status: Defined

Purpose: Define the configuration flow shown after package selection and before the item is added to the cart.

---

# Flow

The customer selects a pricing package, completes the configuration required by that service, reviews the order summary and adds the item to the cart.

Order configuration must occur after pricing selection and before cart creation.

---

# Dynamic Service Fields

Fields are generated from service configuration. A service defines its own required customer inputs, labels, help text, validation rules and display order. Form components must not hardcode service-specific fields.

Examples include a social profile URL, username, post URL, quantity adjustment where allowed, email or service notes.

---

# Supported Field Types

The form architecture must support:

- Text input
- Email input
- URL input
- Number input
- Textarea
- Select
- Radio group
- Checkbox
- Hidden value

Additional field types can be introduced through the field renderer without changing the order flow.

---

# Validation

Validation must run before an item is added to the cart. It must support:

- Required fields
- Type validation
- URL and email format validation
- Minimum and maximum values
- Allowed options
- Service-specific validation rules

Errors appear adjacent to the related field, use clear text and must be announced accessibly.

---

# Components

- `OrderConfigurationSection` тАФ flow container and selected package context
- `DynamicOrderForm` тАФ renders configured service fields
- `DynamicField` тАФ field metadata and shared form behavior
- `FieldRenderer` тАФ maps field types to UI controls
- `ValidationMessage` тАФ field-level validation feedback
- `OrderSummary` тАФ selected package, price and configuration overview
- `AddToCartButton` тАФ validates and creates the cart item

---

# Order Summary

The summary must show the selected service, package, quantity, price, delivery estimate and any eligible discount. It should update as configuration choices affect the order.

---

# Accessibility

Inputs require visible labels, keyboard operation, descriptive help text where needed and accessible validation messaging. The selected package context must remain understandable to screen-reader users.

---

- [ ] Configuration appears after package selection and before cart
- [ ] Service fields are generated from configuration data
- [ ] All listed field types are supported
- [ ] Validation blocks invalid cart additions
- [ ] Errors are clear and associated with fields
- [ ] Components match the defined reusable component list
- [ ] Summary reflects the selected package and configuration
