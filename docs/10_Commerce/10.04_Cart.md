# InstantViral v2

## Cart

Version: 1.0

Status: Defined

Purpose: Define the cart experience for reviewing configured order items before checkout, without collecting payment.

---

# Cart Page Layout

The cart page uses a two-column desktop layout:

- Main column: cart items and coupon form
- Summary column: order totals and checkout action

On smaller screens, the summary follows the item list in a single-column flow.

---

# Components

- `CartPage` — page-level container and cart state boundary
- `CartList` — collection of cart items
- `CartItem` — configured service, package, quantity, price and item actions
- `CartSummary` — subtotal, discount, total and order overview
- `CouponForm` — coupon entry and feedback
- `EmptyCart` — empty-cart guidance and return path
- `CheckoutButton` — continues to checkout

---

# Cart Items

Each cart item must show:

- Service name
- Selected package and quantity
- Relevant configured details
- Delivery estimate
- Item price
- Remove action

Item configuration must remain intact when a customer returns from the cart or checkout flow.

---

# Coupon Handling

The cart provides a coupon form for future coupon validation. The UI must clearly communicate pending, applied, invalid and removed states. Coupon calculations are reflected in the cart summary.

---

# Empty Cart

When no items are present, show `EmptyCart` with a concise explanation and a clear action to continue browsing services.

---

# Persistence

Cart state persists for the current session so configured items remain available during navigation and refreshes within that session. Persistence must not expose sensitive information.

---

# Payment Boundary

The cart does not collect payment details and does not call payment gateways. Its sole checkout action moves the customer to the checkout page.

---

# Accessibility

The page must use accessible headings, announce cart updates, provide keyboard-accessible item actions and ensure totals remain understandable without visual layout.

---

# Acceptance Checklist

- [ ] Desktop uses a two-column cart and summary layout
- [ ] All listed cart components are present
- [ ] Item details and removal are supported
- [ ] Coupon form supports clear feedback states
- [ ] Empty-cart state is available
- [ ] Cart persists for the current session
- [ ] No payment collection or gateway calls occur in cart
