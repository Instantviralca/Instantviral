# InstantViral v2

## Payment Gateway System

Version: 1.0

Status: Architecture Defined

Purpose: Define a provider-independent payment abstraction that can support multiple payment gateways without coupling commerce flows to a specific provider.

---

# Architecture

The payment gateway system exposes a common interface to checkout and order management. Provider adapters implement gateway-specific behavior while the rest of the application works with normalized payment requests, responses and states.

Initial provider targets:

- Stripe
- PayPal

---

# Core Operations

Each provider adapter must support the following operations:

- `createPayment` — initialize a payment for an order
- `verifyPayment` — confirm the provider result and normalized payment state
- `cancelPayment` — cancel an eligible payment attempt

The interface must be extensible for refunds, webhooks and alternative payment methods.

---

# Payment States

Normalized payment states:

- Pending
- Processing
- Paid
- Failed
- Cancelled
- Refunded

State transitions must be explicit, recorded against an order and safe to process repeatedly when provider notifications are retried.

---

# Provider Independence

Checkout and order-management components must not depend on Stripe- or PayPal-specific data structures. Provider identifiers and provider metadata remain inside the payment adapter boundary.

---

# Security Boundaries

- Never expose provider secrets in client code or documentation data.
- Never persist raw card information.
- Verify payment outcomes on a trusted server when integrations are introduced.
- Restrict access to provider credentials and webhook verification secrets.

---

# Current Scope

This document defines architecture only. No provider credentials, payment API calls, SDK initialization or live payment integration is included at this stage.

---

# Future Integrations

The system is designed to support:

- Provider webhooks
- Refund operations
- Payment retries
- Saved payment methods where compliant
- Regional payment methods
- Subscription billing
- Reconciliation and audit reporting
