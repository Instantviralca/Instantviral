# InstantViral v2

## Checkout

Version: 1.0

Status: Defined

Purpose: Define the one-page checkout experience that collects customer details, confirms the order and prepares a future payment handoff.

---

# Checkout Layout

Checkout is a one-page flow. It presents customer information, order review, payment method selection, coupon controls, terms agreement and a final place-order action in a clear sequence.

---

# Components

- `CustomerInformationForm` — customer contact and required billing details
- `OrderSummary` — cart items, discounts and total
- `PaymentMethods` — available payment method choices
- `CouponSection` — coupon entry and applied-coupon state
- `TermsAgreement` — required terms acceptance
- `PlaceOrderButton` — validates the checkout before order creation

---

# Customer Information

The customer information form must collect only the information required to fulfil and support the order. It must provide field labels, inline validation and clear privacy-conscious copy.

---

# Order Summary

The summary must show all cart items, configured service details, prices, discounts and the final total. It must remain visible or easily reachable throughout checkout.

---

# Payment Methods

Payment methods are presented as a UI selection layer only. The design must allow future provider-specific payment options without coupling checkout to a provider.

---

# Coupon Section

Customers can enter, apply and remove coupons from checkout. Applied discounts must update the order summary and total.

---

# Terms Agreement

The customer must explicitly agree to the applicable terms before placing an order. The agreement control must link to the relevant terms and be validated before order submission.

---

# Current Integration Boundary

No payment gateway integration is implemented at this stage. Checkout must not store payment secrets, collect card data directly or make payment-provider API calls.

---

# Accessibility

The one-page flow must support logical keyboard navigation, labeled controls, accessible errors, visible focus states and an understandable order total for assistive technology.

---

# Acceptance Checklist

- [ ] Checkout is a one-page flow
- [ ] Components match the defined reusable component list
- [ ] Customer details, summary, payment-method UI, coupon section and terms agreement are included
- [ ] Terms agreement is required before placing an order
- [ ] Order totals update for coupon changes
- [ ] No payment gateway integration, secrets or API calls exist yet
