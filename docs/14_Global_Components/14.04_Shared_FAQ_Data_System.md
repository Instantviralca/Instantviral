# Document 14.04 — Shared FAQ Data System

Version: 1.0
Status: Production Specification

## Purpose

Create one centralized FAQ data system that can be reused across:

- Homepage
- Main FAQ page
- Service pages
- Contact page
- Checkout support
- Order tracking
- Legal/support pages
- Future Learn articles
- Admin management

The system must prevent duplicated questions, inconsistent answers, and mismatched FAQPage schema.

---

## Objectives

The FAQ system should:

- Store FAQs in one shared data source
- Support categories, platforms, services, and page locations
- Support search and filtering
- Support approval and publication status
- Reuse the exact same answers in UI and schema
- Prevent inactive or hidden FAQs from appearing publicly
- Support future admin CRUD
- Support internal links to services and policies

---

## Service Matching

Service pages should prioritize FAQs in this order:

1. Exact service FAQs
2. Platform FAQs
3. General order/delivery/refund FAQs

Do not show irrelevant FAQs from other services.

---

## Homepage FAQ Selection

Homepage should display 6–8 approved featured FAQs.

Recommended mix:

- No password required
- Delivery timing
- Refill eligibility
- Refund eligibility
- Order tracking
- Supported platforms
- Payment availability
- Customer support

Do not repeat the entire FAQ page on the homepage.

## Schema Rules

FAQPage schema may be generated only when:

- FAQ is Approved
- FAQ is publicly visible
- Question and answer are rendered on the page
- schemaEligible is true
- Answer is not hidden behind inaccessible content
- Content is not duplicated incorrectly

Use the exact rendered question and answer data.

Do not maintain separate schema-only FAQ text.

---

## Schema Scope

Homepage:

Use only the FAQs visibly rendered on the homepage.

Service page:

Use only the FAQs visibly rendered on that exact service page.

Main FAQ page:

Use the full active visible FAQ set rendered on that page.

Contact/support pages:

Generate FAQPage schema only if a visible FAQ section exists and is appropriate.

---

## Duplicate Detection

Prepare helpers to detect:

- Identical questions
- Near-duplicate questions
- Conflicting answers
- Same question assigned to multiple IDs
- Service-specific question duplicated as general FAQ

Do not automatically delete duplicates; report them for review.

---

## Content Quality Rules

FAQs should:

- Answer the question directly
- Use plain English
- Avoid keyword stuffing
- Avoid unsupported guarantees
- Link to policies where needed
- Keep refund/refill wording conditional
- Keep payment methods configuration-driven
- Include YouTube monetization disclaimers where relevant
