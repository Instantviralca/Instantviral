# InstantViral v2

# Document 14.03 — Review & Aggregate Rating Schema Engine

Version: 1.0
Status: Production Specification

## Purpose

Create a centralized schema engine for genuine review and aggregate rating data.

The engine must prevent invalid, misleading, unsupported, or fabricated Review and AggregateRating structured data from being published.

It should work with:

- Homepage
- Service pages
- About page
- Future review pages
- Future admin review management

---

## Objectives

The schema engine should:

- Read only approved public review data
- Validate rating values
- Validate reviewed entities
- Calculate aggregate ratings from eligible reviews
- Prevent schema output when requirements are not met
- Keep visible review content and structured data consistent
- Support service-specific and brand-level review contexts
- Never create rating data from placeholders or marketing copy

---

## Supported Schema Types

Potentially supported when valid:

- Review
- AggregateRating
- Service
- Organization

Do not automatically attach review schema to every page.

---

## Eligibility Rules

Review schema may be generated only when all conditions are true:

- Review is genuine
- Review status is Approved
- Review is publicly visible
- Consent is confirmed
- Numeric rating exists
- Rating is within the supported scale
- Reviewed entity is clearly identified
- Review content is visible on the same page
- Review is relevant to the page entity
- Customer identity display follows consent rules

If any required condition fails, do not output Review schema.

---

## Aggregate Rating Eligibility

AggregateRating may be generated only when:

- At least one eligible numeric review exists
- All included reviews match the reviewed entity
- Aggregate values are calculated from source reviews
- The rating summary is visible on the page
- The review count is accurate
- Hidden, draft, rejected, unrelated, or unrated reviews are excluded

Do not publish aggregate ratings from manually entered values that do not match the review data.

---

## Reviewed Entity Rules

Supported entity contexts:

### Service-Level

Examples:

- Buy Instagram Followers
- Buy TikTok Likes
- Buy Facebook Page Likes
- Buy YouTube Views

Only include reviews linked to the exact service.

### Platform-Level

Examples:

- Instagram services
- TikTok services

Use only when the page clearly represents that platform grouping and the visible reviews match it.

### Organization-Level

Use general InstantViral reviews only where appropriate and supported.

Do not use service reviews as organization reviews without clear relevance.

---

## Rating Scale

Default:

- worstRating: 1
- bestRating: 5

Requirements:

- ratingValue must be numeric
- reviewCount must be an integer
- Values must match the approved review data
- Do not round deceptively
- Use a consistent decimal policy

Recommended aggregate display:

One decimal place where needed.

---

## Review Schema Fields

Each eligible Review may include:

- `@type`
- `itemReviewed`
- `author`
- `datePublished`
- `reviewBody`
- `reviewRating`
- `publisher` when appropriate

Do not include internal order IDs, customer email, moderation notes, or private information.

---

## Aggregate Rating Fields

Supported fields:

- `@type`
- `ratingValue`
- `reviewCount`
- `bestRating`
- `worstRating`

The reviewed entity must match the visible page entity.

---

## Required Helpers

Create reusable helpers:

- `getSchemaEligibleReviews()`
- `canGenerateReviewSchema()`
- `canGenerateAggregateRating()`
- `calculateAggregateRating()`
- `buildReviewSchema()`
- `buildAggregateRatingSchema()`
- `validateRatingValue()`
- `validateReviewedEntity()`
- `getPublicReviewAuthor()`
- `sanitizeReviewForSchema()`

---

## Validation Rules

Reject schema when:

- No approved review exists
- Review is hidden
- Consent is missing
- Rating is missing
- Rating is outside range
- Review text is not visible on page
- Review belongs to another service
- Review entity is ambiguous
- Aggregate count does not match source data
- Placeholder data is detected
- Customer identity is not approved for publication

---

## Placeholder Detection

The system should guard against development placeholders.

Examples to reject:

- Test User
- John Doe
- Sample Review
- Lorem ipsum
- Placeholder testimonial
- Demo rating
- Hardcoded 5.0 rating
- Hardcoded 100 reviews

Do not rely only on keyword detection; use review status and source validation.

---

## Page Integration

### Homepage

Use organization-level reviews only when genuine and visibly rendered.

### Service Pages

Use exact service reviews only for service-level Review or AggregateRating schema.

### About Page

Use organization-level reviews only when visible.

### Pages Without Reviews

Do not output Review or AggregateRating schema.

---

## Visible Content Consistency

Structured data must match visible content.

If a review is removed from the page:

- Remove it from schema
- Recalculate aggregate rating
- Update review count

If the review section is hidden:

- Do not output review schema

---

## Admin Controls

Prepare admin settings for:

- Enable review schema
- Enable aggregate rating schema
- Preview eligible reviews
- Preview calculated rating
- View excluded reviews and reasons
- Validate entity match
- Rebuild schema preview

Do not allow manual override of calculated rating values in Version 1.

---

## Logging

Prepare logging for:

- Schema generated
- Schema skipped
- Review excluded
- Invalid rating
- Missing consent
- Entity mismatch
- Placeholder detected
- Aggregate recalculated

Logs should be internal only.

---

## Testing Requirements

Create tests for:

- Valid service review
- Invalid hidden review
- Missing consent
- Missing rating
- Out-of-range rating
- Mixed service reviews
- Aggregate calculation
- Empty review set
- Hidden review section
- Placeholder review
- Author privacy
- Decimal rating handling

---

## Security and Privacy

- Never expose internal order IDs
- Never expose customer email
- Never expose moderation notes
- Respect approved public display name
- Sanitize review text
- Prevent script injection
- Do not include hidden personal information in JSON-LD

---

## Performance

- Calculate schema server-side where possible
- Cache safe aggregate results when appropriate
- Recalculate when approved review data changes
- Avoid client-only schema generation
- Keep JSON-LD output minimal and valid
