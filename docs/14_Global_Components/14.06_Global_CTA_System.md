# InstantViral v2

# Document 14.06 — Global CTA System

Version: 1.0
Status: Production Specification

## Purpose

Create a reusable Call-to-Action (CTA) system that is shared across the entire website.

Supported pages:

- Homepage
- Service pages
- About
- Contact
- FAQ
- Legal pages
- Order Tracking
- Checkout
- Future Learn Center

---

## Objectives

- Consistent CTAs across the site
- Configuration-driven content
- Platform and service specific variants
- Support A/B testing in the future
- Analytics-ready
- Accessible
- No duplicated CTA copy

---

## CTA Types

### Primary
- Browse Services
- Buy Now
- Get Started

### Secondary
- Contact Support
- Track Your Order
- View Pricing

### Cross-sell
- Related Services
- Explore More Services

### Informational
- Read FAQ
- Refund Policy
- Contact Us

---

## CTA Data Model

Each CTA supports:

- id
- title
- description
- buttonLabel
- destination
- variant
- icon (optional)
- platform (optional)
- serviceSlug (optional)
- pageLocations
- active
- order

---

## Required Components

- CTASection
- CTAHero
- CTAInline
- CTAFooter
- CTACard
- CTAButtonGroup

---

## Business Rules

- Do not hardcode CTA copy.
- Read all CTA content from the shared content layer.
- Hide inactive CTAs.
- Validate destinations before rendering.
- Support internal links only in Version 1.

---

## Accessibility

- Clear button labels
- Keyboard accessible
- Visible focus states
- Minimum 44px touch targets

---

## Analytics

Prepare events:

- cta_view
- cta_click
- cross_sell_click
- support_cta_click
- checkout_cta_click

---

## Acceptance Checklist

- Shared CTA registry created
- Components reusable
- Content layer implemented
- Responsive
- Accessible
- Typecheck passes
- Lint passes
- Production build passes
