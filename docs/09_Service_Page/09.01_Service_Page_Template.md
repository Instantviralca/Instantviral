# InstantViral v2

# Document 09.01 — Service Page Template

Version: 1.0

Status: Production Specification

---

# Purpose

This document defines the standard layout for every commercial service page on InstantViral.

All service pages must use this template regardless of platform.

Examples:

- `/instagram/buy-instagram-followers`
- `/instagram/buy-instagram-likes`
- `/tiktok/buy-tiktok-followers`
- `/youtube/buy-youtube-subscribers`
- `/facebook/buy-facebook-followers`

---

# Primary Goals

- Explain the service clearly.
- Present pricing packages early.
- Build trust.
- Answer common questions.
- Encourage conversion.

---

# Standard Layout

1. Hero
2. Pricing Packages ⭐
3. Key Benefits
4. Features
5. How It Works
6. Customer Reviews
7. FAQ
8. Related Services
9. Final CTA

---

# Pricing Packages

The pricing section is the main conversion block.

Each package card should include:

- Package name
- Quantity
- Estimated delivery
- Price
- Savings (optional, if genuine)
- Features included
- Primary CTA

The pricing data must come from the data layer or API.

No hardcoded package values.

---

# Desktop Layout

- Maximum width: 1280px
- 12-column grid
- Consistent spacing between sections
- Sticky order summary only if introduced later (not required initially)

---

# Tablet Layout

- Responsive grids
- Two-column package layout where appropriate

---

# Mobile Layout

- Single-column layout
- Pricing cards stacked vertically
- Large touch targets

---

# CTA Strategy

Primary CTA:

- Continue / Order Now

Secondary CTA:

- Learn More or View Related Services

CTA labels should remain configurable through the content layer.

---

# Internal Linking

Every page links to:

- Parent platform page
- Related services
- Relevant Learn articles

Breadcrumbs should be visible near the top of the page.

---

# SEO Placement

Above the fold:

- H1
- Intro
- Primary CTA

Below:

- Pricing
- Supporting content
- FAQ
- Related Services

Maintain one H1 only.

---

# Accessibility

- Semantic landmarks
- Keyboard-friendly package cards
- Visible focus states
- Accessible accordions
- Proper heading hierarchy

---

# Performance

- Lazy-load below-the-fold media
- Optimize images with next/image
- Keep pricing interactions lightweight

---

# Analytics

Track:

- Package selection
- CTA clicks
- Scroll depth
- FAQ interactions
- Related service clicks
- Checkout starts

---

# Component Composition

```
ServicePage
├── ServiceHero
├── PricingPackages
├── BenefitsSection
├── FeaturesSection
├── HowItWorksSection
├── ReviewsSection
├── FAQSection
├── RelatedServicesSection
└── FinalCTASection
```

Each section must remain independent and reusable.
