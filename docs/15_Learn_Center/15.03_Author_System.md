# Document 15.03 — Author System

Version: 1.0
Status: Production Specification

## Purpose

Create a centralized Author System for the Learn Center.

Goals:
- Support E-E-A-T
- Reusable author profiles
- Author pages
- Person schema
- SEO-ready architecture
- Future multi-author support

## Public Routes

- /authors
- /authors/[slug]

## Core Rules

- All author data comes from one shared registry.
- Never hardcode author details in articles.
- Never invent credentials or qualifications.
- Hide inactive authors.

## Author Data Model

- id
- slug
- name
- role
- bio
- avatar
- coverImage
- website

- expertise

## Accessibility

- Semantic headings
- Keyboard navigation
- Accessible images

## Security

Never expose:
- Private email
- Internal notes
- Draft articles
- Admin-only metadata
