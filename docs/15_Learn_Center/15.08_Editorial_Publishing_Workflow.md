# Document 15.08 — Editorial Publishing Workflow

Version: 1.0
Status: Production Specification

## Purpose

Define the complete editorial workflow for Learn Center articles from draft to archive.

## Workflow

Draft
→ In Review
→ Approved
→ Scheduled (optional)
→ Published
→ Updated
→ Archived

Only Published articles are publicly accessible and indexable.

## Article Status Model

Support:

- Draft
- In Review
- Approved
- Scheduled
- Published
- Updated
- Archived

Reject invalid transitions.

## Required Fields

- status
- publishedAt
- updatedAt
- scheduledAt
- approvedBy
- reviewedBy
- reviewNotes
- seoReviewed
- contentReviewed
- lastEditorialUpdate

Internal editorial fields must never be exposed publicly.

## Publishing Validation

Before publishing verify:

- H1 exists
- Meta title
- Meta description
- Canonical URL
- Author assigned
- Featured image
- Category assigned
- Valid tags
- Reading time
- Related articles
- Internal links
- Valid schema
- Accessibility checks
- No broken internal or external links

Publishing must be blocked if validation fails.

## Scheduling

Support future publication.

Requirements:

- Future publish date
- Automatic publication
- Automatic sitemap inclusion
- Automatic robots update
- Automatic metadata activation

Scheduled articles remain private until publication.

## Updating

When updating published articles:

- Preserve datePublished
- Update dateModified
- Refresh sitemap lastModified
- Refresh caches
- Recalculate related articles if needed

## Archiving

Archived articles:

- Removed from Learn listings
- Removed from sitemap
- noindex
- Excluded from search
- Excluded from related articles

Prepare redirect support for the future.

## Security

Never expose:

- Review notes
- Internal comments
- Reviewer-only metadata
- Draft content
- Scheduled content

- Robots update
